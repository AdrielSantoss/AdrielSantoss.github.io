Entendendo a Arquitetura REST

10/11/2025 - Projeto (Github)
Node Typescript Arquitetura Rest Restful Fastify Prisma ORM

Entre 2019 e 2020, eu desenvolvia diversas APIs REST em Node.js, utilizando o Express para criar o servidor HTTP e o ORM Knex para modelar as tabelas do banco de dados.

Desde então, acabei me afastando um pouco do ecossistema Node, e nesse meio tempo surgiram (ou se popularizaram) vários frameworks e bibliotecas modernas — como o Fastify, que oferece uma performance excelente na criação de servidores web, e o ORM Prisma, que traz uma linguagem de modelagem intuitiva, configuração simples e ótimo desempenho.

Naquela época, eu ainda não utilizava TypeScript e recorria ao Jest para implementar testes unitários e de integração nas APIs.

Hoje percebo que o Knex tem recebido poucas atualizações e parece ter ficado atrás do Prisma em termos de recursos e praticidade. O Express continua sendo uma ótima escolha, mas o Fastify vem se destacando em performance e escalabilidade. Já o Jest, por não oferecer suporte nativo a TypeScript em APIs Node, tem perdido espaço para ferramentas mais modernas como o Vitest.

Na verdade isso é um processo natural: bibliotecas surgem, se popularizam, e depois de um tempo morrem, perdem espaço para uma nova biblioteca que dominou a área. Por isso, eu não gosto de escrever sobre bibliotecas e frameworks, pois esse conteúdo costuma se defazar muito rápido. Nesse blog, o objetivo é falar sobre conceitos de arquitetura REST, o foco não é as novas bibliotecas, elas foram apenas o gatiho que me motivou a fazer esse projeto/estudo.

A comunidade JavaScript/Node é extremamente ativa — as tecnologias evoluem rápido e, na maioria das vezes, para melhor. Por isso, recentemente decidi revisitar meus conhecimentos de arquitetura REST na prática, criando um servidor HTTP com Node + TypeScript + Fastify + Prisma + Vitest, entre outras ferramentas.

Além de relembrar conceitos, quero aproveitar essa oportunidade para pesquisar e aprender coisas novas.
