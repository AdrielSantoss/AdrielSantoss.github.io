Observabilidade vs Monitoramento

Podemos entender o que é observabilidade comparando com monitoramento, este é o tipo de confusão mais comum quando trata-se de entender observabilidade. A observabilidade utiliza o monitoramento como base, mas eles são conceitos distintos.

Começando pelo monitoramento. De forma resumida, monitoramento é a prática de acompanhar os sinais de um sistema e definir alertas para situações conhecidas. Na prática, o desenvolvedor estabelece limites ou condições específicas e quando o sistema atinge esses critérios, um alerta é acionado.

Basicamente, monitoramento responde uma pergunta simples como: "O sistema está saudável?". Para responder essa pergunta é configurado alertas para quando algumas métricas conhecidas saem do normal. Na prática, no monitoramento o sistema já "conhece" o que pode dar errado, por isso ele fica monitorando esses pontos e emitindo alerta.

Já a observabilidade tenta responder o "Por que isso está acontecendo?", analisando os dados (logs, métricas e traces), o sistema tenta achar uma resposta para um problema totalmente desconhecido.

Para entender com clareza, imaginamos o seguinte cenário:


1. Usuário Osvaldo, abre o sistema de PDV e tenta efetuar uma venda no PIX.
2. Mas ao tentar gerar o QRCODE do PIX, ocorre um TIMEOUT na comunicação com o gateway de pagamento.

Nesse cenário, o monitoramento emitirá um alerta do tipo:
"Erro na comunicação com o gateway de pagamento: tempo de resposta excedido."

Perceba que nesse exemplo, o dashboard de monitoramento não consegue responder algumas perguntas como:
1. O problema é no meu código ou no provedor? 
    (O Trace mostraria exatamente em qual milissegundo a requisição saiu da sua rede e "morreu" na rede do gateway).

2. É um problema geral ou específico? 
    (O erro só acontece com o Osvaldo porque ele está usando uma versão antiga do PDV ou um modelo específico de maquininha?)

3. Qual foi a jornada desse erro? 
    (A requisição passou pelo serviço de Vendas, depois pelo serviço de Pagamentos, mas travou no serviço de Autenticação?)
Etc...

Um dashboard de monitoramento exibe gráficos da saúde e desempenho do sistema, isso até pode transmitir uma sensação de "segurança", mas quando um problema complexo ocorre em produção, esse dashboard não vai te responder perguntas cruciais para entender o problema. Com as indicações do gráfico e alertas de monitoramento, já sabemos o que aconteceu (erro na comunicação com o gateway), mas sem os pilares da observabilidade vai ser dificil entender o porquê aconteceu.

Estudar o passado é compreender o presente
