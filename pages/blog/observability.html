Observabilidade vs Monitoramento

Podemos entender o que é observabilidade comparando com monitoramento, este é o tipo de confusão mais comum quando trata-se de entender observabilidade. Além disso, é necessário saber sobre o conceito de telemetria.

Começando pelo monitoramento. De forma resumida, monitoramento é a prática de acompanhar os sinais de um sistema e definir alertas para situações conhecidas. Na prática, o desenvolvedor estabelece limites ou condições específicas e quando o sistema atinge esses critérios, um alerta é acionado.

O monitoramento normalmente depende da telemetria, responsável pela coleta de dados do sistema. Essas informações refletem o estado atual da aplicação e podem ser registradas em bancos de dados, sistemas de logs ou até mesmo em arquivos simples.

Para entender melhor, vamos visualizar e analisar alguns exemplos na prática, começando por este: 

using System;
using Serilog;

class Program
{
    static void Main(string[] args)
    {
        // Configuração do Serilog para salvar em arquivo
        Log.Logger = new LoggerConfiguration()
            .WriteTo.File("logs/log.txt", rollingInterval: RollingInterval.Day)
            .CreateLogger();

        try
        {
            // Simulando um erro
            MetodoComErro();
        }
        catch (Exception ex)
        {
            // Registrando erro com stacktrace
            Log.Error(ex, "Ocorreu um erro durante a execução");
        }
        finally
        {
            Log.CloseAndFlush();
        }
    }

    static void MetodoComErro()
    {
        throw new InvalidOperationException("Erro de teste");
    }
}

Esse é um exemplo clássico de telemetria com Serilog (biblioteca de logging do C#). Isso não é monitoramento, isso é telemetria simples, o sistema não está reagindo ao erro, ele está apenas registrando o erro.

Entenda que todo monitoramento é um processo de detecção e reação, o sistema reage a uma condição, normalmente emitindo alertas/avisos. O exemplo acima não possui nada disso.
