Sobre meus conhecimentos Técnicos


Front-end

Para desenvolvimento de sistemas front-end, priorizo utilizar o superconjunto de Javascript, chamado de Typescript, sempre que possivel. Acredito que uma linguagem fortemente tipada é mais eficiente e benefica no processo de desenvolvimento de um projeto complexo.

Com a linguagem TypeScript/Javascript, utilizo os principais frameworks de front-end modernos, frameworks que possuem o conceito de Single Page Application, como base. Os principais que estão presentes em meus projetos são: React JS, Angular e Vue JS.

Back-end

Para sistemas back-end, crio API's com as seguintes linguagens: C#, Javascript e Python. Com essas linguagens utilizo bibliotecas para criar servidores HTTP, declarando os endpoints (controllers) e services. Eu uso ORM para manipulação e mapeamento de banco de dados.

Para modelagem de banco de dados relacionais, utilizo ORM como por exemplo o Entity Framework Core (C#) ou KNEX (NodeJS/Javascript). Nessa modelagem, gero as MIGRATIONS dos modelos que mapeam os relacionamento entre as tabelas (1 para 1, N para 1 ou N para N).

Banco de dados

Dependendo da necessidade e da arquitetura do projeto, escolho entre banco de dados RELACIONAL ou NÃO-RELACIONAL. Aqui estão algun que costumo utilizar em meus projetos: MySQL, PostgreSQL, SQLite, Firebird, MongoDB e Firebase.

Mobile

Para aplicativos móveis, uso frameworks como Flutter e React Native. Em aplicativos móveis, geralmente implemento integrações com APIs REST, ou plataformas para armazenamento de dados em nuvem, como o Firebase da Google. Além disso, utilizo os recursos nativos de um smartphone, como localização, câmera e etc.
