Descrição


Este projeto, trata-se de um aplicativo Mobile para controle de finanças de empresas.
O objetivo desta aplicação é exibir em forma de valores, gráficos e percentuais os resultados e métricas de uma ou mais empresas em determinados períodos de tempos.

Este projeto, foi o primeiro produto Mobile de uma empresa, isto é, fui o desenvolvedor encarregado de criar a primeira experiência da empresa em uma nova plataforma. A linguagem, framework e banco de dados também foram tecnologias inovadoras dentro da empresa. A linguagem é o Dart, o framework é o Flutter e o banco de dados é o Firebase.

Este projeto possui diversos sistemas e integrações que compõem suas funcionalidades, segue abaixo alguns pontos de caractaristicas técnicas do projeto:

1. Fui o responsável por publicar o aplicativo nas duas principais lojas de aplicativos Mobile atualmente, App Store para IOS e Play Store para Android.

2. Somente os clientes da empresa, podem criar contas e acessar o aplicativo, a criação de contas é vinculada a uma integração com um sistema web, este sistema web armazena os dados das contas de usuário dos clientes, sendo assim, para um usuário autenticar no aplicativo, é necessário criar uma conta no sistema web, para entender melhor esse fluxo, veja o diagrama abaixo:


3. O banco de dados que é utilizado no Aplicativo, chama-se Cloud Firestore que é um banco NOSQL da plataforma Firebase. Para que o Firestore possua os dados de cada cliente, como valores de vendas, percentuais por empresas, gráficos e etc, é realizado uma migração de dados, sendo executado uma operação chamada de ETL (Transfom, Extract, Load), essa operação, consiste em extrair dados de uma fonte, e em seguida transformar ou adaptar esses dados, para serem inseridos em uma nova base de dados. Sendo assim, os dados que são inseridos no Firestore, são extraidos de uma base de dados dos clientes de outros sistemas que eles já utilizam, através de um script que é encarregado de fazer a operação ETL.


4. O aplicativo possui um sistema de Notificações, este sistema faz parte da plataforma Firebase e chama-se Firebase Cloud Message (FCM). Este sistema de notificações é integrado com uma Web Api, que gerencia e armazena toda notificação que é disparada. Sendo assim, qualquer sistema que queira disparar uma notificação no aplicativo Mobile, basta enviar uma requisição HTTP para esta Web Api (enviando os parametros necessários), e a Web Api por sua vez, se comunicará com o FCM e a notificação será entregue para os dispositivos corretos.
