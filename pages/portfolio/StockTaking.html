StockTaking App

Flutter Dart Kotlin Android

Descrição

Neste projeto, foi desenvolvido um aplicativo mobile exclusivo para plataforma Android, cujo o seu objetivo é integrar com um hardware especializado nas funções de leitura de códigos de barras e QR Codes.

Este projeto surgiu a partir de uma parceria de duas empresas, a primeira, sendo a desenvolvedora do hardware, e a segunda, sendo a desenvolvedora do software. Neste projeto, fui o desenvolvedor responsável por atuar na segunda empresa, criando o software que iria integrar com o hardware.

A programação dessa aplicação foi seguindo as especificações e documentações do Hardware, ou seja, além de programar, obtive experiências comuns na rotina de um programador, tais como: ler documentação, reportar bugs, sugerir melhorias e manter contato ativo com o suporte técnico da empresa parceira.

Este aplicativo, juntamente com o dispositivo, são uma ferramenta para o trabalho de contagem de estoque, este aplicativo possui integração com o sistema Desktop que os clientes utilizam para inserir essa contagem. Sendo assim, além de facilitar a contagem em lote de inúmeros produtos de forma rápida, os dados coletados na contagem serão enviados para o banco de dados do cliente e estarão disponiveis para visualização e revisão no sistema Desktop que já utilizam em suas lojas/empresas.

Autenticação

Para utilizar o StockTaking App, é necessário realizar LOGIN. A primeira tela do app, após a animação de entrada, é a tela de autenticação, onde o usuário pode inserir informações como USUÁRIO e SENHA, Essa conta de acesso o usuário JÁ possui, pois as contas são integradas com o sistem Desktop ERP.

A autenticação deste aplicativo é integrada com o sistema Desktop ERP, que os clientes já utilizam, isto é, se o funcionário da loja que está encarregado de fazer a contagem, possui conta no sistema Desktop, logo, ele pode autenticar-se com esta mesma conta no App Mobile, sem a necesidade de criação de uma nova conta.

Sendo assim, para essa integração de contas funcionar, o App mobile deverá acessar o mesmo banco de dados que o Sistema Desktop acessa para autenticar usuários. O acesso deste banco é liberado atráves de um Sistema WEB, ou seja, o App comunica-se via requisição HTTP com este sistema web para enviar informações dos usuarios e receber o token de autenticação. Integração com o Hardware

Para integrar com o Hardware, isto é, para ligar/desligar o leitor do aparelho a partir do aplicativo, foi necessário criar funções na linguagem nativa para Android com Kotlin, e essas funções serão executadas atráves de eventos no Flutter, estes eventos fazem parte do sistema de METHOD CHANNELS do Flutter.

A partir do Client (código dart com Flutter) é possivel executar chamadas de funções que foram criadas em lianguagens nativas como: Kotlin/Java, Swift/Objective-C no iOS e etc. Para entender melhor este fluxo, veja o diagrama abaixo disponibilizado na documentação do Flutter:


É necessário criar funções em Kotlin, pois, para manipular o leitor do Aparelho, foi utilizado funções do SDK de integração com o Sistema Operacional do Hardware, ou seja, a empresa criadora do Hardware, desenvolveu um SDK de integração, porém, esse SDK só estava disponivel para Kotlin/Java. Sendo assim, foi necessário codificar em linguagem nativa, para conseguir realizar a importação e utilização do SDK para manipular o leitor.

As duas principais funções utilizadas do SDK foram: a função de INICIALIZAÇÃO e a função de INTERRUPÇÃO do leitor, sendo assim, atráves de eventos emitidos pelos CHANNELS do Flutter os métodos Kotlin eram executados e o app conseguia controlar quando o leitor deveria ser ativado e desativado.

Gráficos, Informações e Conferência

Toda vez que um codigo de barras ou QR Code de um produto é lido pelo aparelho, é exibido no aplicativo um Card com as informações do produto, como Nome e Saldo em Estoque e Quantidade (contagem). Para cada evento de leitura de código de barras, é realizado uma requisição HTTP para o sistema WEB, esta requisição do tipo (verbo http) GET, envia como parâmetro (query-params) o código de barras do produto, e recebe como Response as informações do produto como Nome, Saldo em estoque e etc. Tornando assim, a contagem algo dinâmico, enquanto o usuário realiza o SCAN dos produtos, o App já "consulta" o banco de dados e lhe mostra as informações do mesmo.

Após o Scan, para que as informações do produto esteja correta no App, é necessário que aquele produto cujo o código de barras é igual ao da leitura, esteja presente no banco de dados. Se ao realizar a leitura de um produto e o mesmo NÃO ser encontrado no banco de dados, o App sinalizará que o produto lido não está cadastrado na base de dados, deixando o card em coloração VERMELHA.

Em uma operação de contagem de estoque é muito comum alguns produtos que estão sendo contados não sejam encontrados no banco de dados, seja porque o produto é novo ou o mesmo nunca foi cadastrado no Sistema Desktop. Também, é muito comum, os valores da contagem de alguns produtos não serem iguais ao Saldo de Estoque do produto, por exemplo: um óculos de sol, cujo seu valor de estoque no sistema Desktop é de 25 unidades, e na contagem, o valor obtido foi 24 unidades. O aplicativo separa essas situações de contagens e exibe todas elas em uma tela de finalização.
